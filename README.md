# AlertOps Investigator Demo API

This is the separate workload used to demonstrate the AlertOps AI Investigator.
It contains a deliberately broken pricing function, so the first GitHub Actions
run fails and sends an event to AlertOps.

## Before pushing

Create a new GitHub repository for this directory. In that repository, add the
following Actions secret:

- `ALERTOPS_INBOUND_URL`: URL generated by the AlertOps Custom Inbound API
  Integration.

The workflow sends these JSON fields: `source`, `status`, `repository`,
`workflow`, `runId`, `commit`, `branch`, `runUrl`, and `message`. Configure the
AlertOps inbound template to map them. Use `repository` as part of the alert
subject and use a stable alias based on the repository/workflow for recovery
correlation.

## Demo sequence

1. Push the intentionally broken implementation.
2. The test expects a 20% discount on 100 to equal 80; the code returns 120.
3. GitHub Actions fails and sends the failure payload to AlertOps.
4. After reviewing the AI report, change `+` to `-` in
   `demo_app/pricing.py`.
5. Push the fix. Tests pass and the workflow sends the recovery payload.
